    Suck your subscription into PWM

    Two environment variables are all it takes. Point your AI client at the PWM gateway and every call runs through — earning you PWM tokens while still using Claude or ChatGPT normally.

    👤 You (consumer) — this page

    Set ANTHROPIC_BASE_URL to route your Claude Code through the network. Your pwm_ key identifies your account — your real Anthropic key stays untouched.

    🖥 Providers — exchange node

    Providers run a local node that holds their real ANTHROPIC_API_KEY. That key never leaves their machine — they earn PWM for each request their node fulfils for consumers like you.

    a

    Get your PWM API key

    Sign in, go to your profile, and generate a pwm_ key. It's free.

    Get API key →
    b

    🤖 Claude Code — point it at the PWM gateway

    Add these two lines to your shell profile (~/.bashrc or ~/.zshrc), then open a new terminal. Claude Code continues to work exactly as before — but each call earns PWM.

    export ANTHROPIC_BASE_URL=https://physicsworldmodel.org/proxy/anthropic
export ANTHROPIC_AUTH_TOKEN=pwm_your_key_here

    Already use ANTHROPIC_API_KEY? Keep it — the gateway only needs ANTHROPIC_AUTH_TOKEN to identify your PWM account.

    c

    💬 Codex / OpenAI — same idea

    Works for the Codex CLI, the OpenAI Python SDK, or any OpenAI-compatible client.

    export OPENAI_BASE_URL=https://physicsworldmodel.org/proxy/openai
export OPENAI_AUTH_TOKEN=pwm_your_key_here
    ⬢

    What happens on each call

    Your request reaches Anthropic / OpenAI as normal. Simultaneously, the gateway records the usage and credits a fixed quantity of PWM tokens to your wallet — usually within seconds. The token amount is set by the protocol's emission equation, not by us.

    You are effectively sucking subscription spend into a permanent, appreciating asset.

    Why your PWM tokens grow in value

    This is not speculation — it follows directly from the protocol's mechanics.

    🔒

    Fixed emission quantity

    Each unit of AI usage through the gateway mints a fixed, predetermined quantity of PWM tokens — set by the emission equation in the white paper. The protocol cannot mint more just because demand rises.

    📈

    Growing network usage

    As more researchers, developers, and institutions route usage through the gateway, total demand for PWM grows — but the supply curve doesn't change. More demand × fixed supply = higher price per token.

    ⏰

    Earlier = more valuable

    The emission equation is Zeno-weighted: early contributions earn more PWM per unit of usage than later ones. Absorbing now — while the network is small — locks in the most favorable conversion rate.

    🔁

    Every call adds value invisibly

    You don't see it happen, but each API call through the network contributes to the protocol's usage record. That record is what backs the token's value — real throughput, not hype.

    📐

    The math (simplified)

    Token value ≈ total usage × price_per_call / circulating_supply. Because circulating_supply grows on a fixed schedule while total usage grows with the network, the ratio — and therefore your tokens' purchasing power — trends upward over time.

    Read the full emission equation in the white paper →
